## Vision

### Compressed CEO statement

Fixo 是 HMLS-family 的消费端 AI 前脸：当 HMLS mobile mechanic 业务不能接的活、当其他 mechanic shop
不接的活、当 ChatGPT/Reddit 答不到的位置 — fixo 承接。**fixo 不是独立 B2C 项目**，是已运营 mechanic
业务的 overflow + 自助前端。这是 12-month 形态；这次 30 天 plan 是它的最小可验证版本。

```

─────────────────             ─────────────────                ─────────────────

                            + Reddit timing                  拒接/overflow 出口

                            (并行 ship+validate)              都把"做不了的活"丢到 fixo
                                                              B2B2C $30-100/shop/mo

zero demand evidence        validated wedge signal           fixo embedded distribution
```

## Accepted Scope（变成 plan 主体的内容）

**Week 1**（并行三轨）:

- **轨 A — 工程**：5 个 SEO 落地页 `/obd/[code]` (从 Top 50 OBD 码筛 5 个最高 search volume 起步)
- **轨 B — HMLS overlay**：在 HMLS 拒接邮件模板 / booking-form rejection page / 已预约满 SMS 加 fixo
  CTA
- **轨 C — Validation**（用户自跑）：90 分钟巡 r/MechanicAdvice，20 个 post 表格

**Week 2**:

- 评估 Week 1 信号
- 加 EEAT block 到 5 SEO 页（每页加 "Reviewed by HMLS mechanics, [city]" + Yelp/Google 链接）
- TikTok creator outreach 开始（找 2-3 个 10K-50K 粉车内容 creator）
- Reddit 手答 5/天 持续

**Week 3**:

- 如果信号 OK，扩容到 30 SEO 页
- TikTok creator videos 上线
- 启动 funnel tracking (GSC + GA4)

**Week 4**:

- 30-day funnel 报告
- 续投 / 切 wedge / pivot 决策

**Explicit Kill Criteria（D5 accepted）**:

- **Week 1 末**：HMLS rejection-flow CTR < 5% → 停 SEO 扩容、回到 dogfood-only 模式
- **Week 2 末**：Reddit observation 显示 ChatGPT 在 ≥80% 紧急场景下达到 fixo 同等输出 → 切 wedge 到
  Accessibility 或 Price-trust
- **Week 3 末**：所有 SEO 页 in GSC 0 impressions（不是 0 clicks，是 0 impressions） → 停 TikTok
  creator spend、切回 dogfood-only
